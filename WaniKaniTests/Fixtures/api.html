Making Requests

Requests are done by URL. Below is the general format of the URL: 

http://www.wanikani.com/api/user/{USER_API_KEY}/{RESOURCE}/{OPTIONAL_ARGUMENT}

The {USER_API_KEY} is a 32 character key that is the user's identifier. {USER_API_KEY} can be generated, regenerated, and viewed in the user's settings. Your API key is 12345. Examples throughout the page will be displayed utilizing your information.

{RESOURCE} is the type of resource you wish to request. The resources available can be found further down in the document. Alternatively, a list of links of each resource can be found at the top of the page.

Some {RESOURCE} accept an optional argument or {OPTIONAL_ARGUMENT}.

Requests are arbitrary throttled to 100 per hour. If the throttle limit is exceeded, the following response will be sent: 

403 Forbidden (Rate Limit Exceeded)

Although the throttle limit is generous, it may be a good idea to cache the request for a reasonable amount of time. Many of the resources you can request do not get updated frequently.
